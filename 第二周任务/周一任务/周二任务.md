💻 周二：通道（Channel）与 Goroutine 通信实战
题目：并发文件处理工作池

目标：掌握通道（Channel）的基本用法，实现 Goroutine 间的安全通信，构建一个可控制并发度的工作池。

## 需求说明

基于周一学习的 Goroutine 和 WaitGroup 基础，今天将引入 Go 并发编程的另一个核心组件：**通道（Channel）**。通道是 Goroutine 之间进行通信和数据交换的安全管道。

请实现一个**并发文件处理工作池**，该程序能够：

### 1. 准备工作
- 创建 200 个测试文件，文件名格式为 `data_001.txt` 到 `data_200.txt`
- 每个文件包含随机生成的 1-100 行文本内容，每行是一个随机的整数（范围 1-1000）

### 2. 工作池设计
- 使用**缓冲通道**创建一个工作池，支持同时处理 10 个文件的并发上限
- 主程序将所有文件路径发送到任务通道（jobs channel）
- 启动固定数量的工作 Goroutine（例如 10 个），每个 Goroutine 从通道接收文件路径进行处理
- 使用 `sync.WaitGroup` 等待所有工作 Goroutine 完成任务

### 3. 文件处理逻辑
每个工作 Goroutine 需要执行以下操作：
1. 读取文件内容
2. 统计文件中的总行数
3. 计算文件中所有数字的总和
4. 将处理结果（文件名、行数、总和）发送到结果通道（results channel）

### 4. 结果收集与展示
- 主程序从结果通道接收所有处理结果
- 按照文件名排序后显示每个文件的统计信息
- 计算并显示总体统计数据：
  - 处理的总文件数
  - 所有文件的总行数
  - 所有数字的总和
  - 整个处理过程的耗时

### 5. 清理工作
- 处理完成后删除所有测试文件
- 使用 `defer` 确保异常情况下也能清理资源

## 技术要点

### 通道（Channel）操作
```go
// 创建缓冲通道
jobs := make(chan string, 100)      // 任务通道
results := make(chan Result, 100)   // 结果通道

// 发送数据到通道
jobs <- filePath

// 从通道接收数据
filePath := <-jobs

// 关闭通道（表示不再发送数据）
close(jobs)
```

### 工作 Goroutine 模式
```go
for i := 0; i < workerCount; i++ {
    go func(workerID int) {
        defer wg.Done()
        for job := range jobs {  // 自动从通道接收，直到通道关闭
            // 处理任务
            result := processFile(job)
            results <- result
        }
    }(i)
}
```

### 结果收集模式
```go
// 启动一个单独的 Goroutine 等待所有工作完成
go func() {
    wg.Wait()
    close(results)  // 所有工作完成后关闭结果通道
}()

// 主程序收集结果
var allResults []Result
for result := range results {  // 自动接收，直到通道关闭
    allResults = append(allResults, result)
}
```

## 预期学习成果

1. **理解通道的本质**：通道是类型安全的 FIFO 队列，用于 Goroutine 间通信
2. **掌握缓冲 vs 非缓冲通道**：缓冲通道允许异步发送，非缓冲通道要求同步收发
3. **学会工作池模式**：控制并发度，避免资源耗尽
4. **实践通道关闭机制**：正确的通道关闭时机和方式
5. **理解 select 语句**（可选扩展）：处理多个通道操作

## 扩展挑战（可选）

1. 使用 `select` 语句为工作池添加超时机制
2. 实现优雅关闭：接收到中断信号（Ctrl+C）时逐步停止工作池
3. 添加错误处理：当文件处理失败时，将错误信息通过错误通道传回主程序
4. 实现进度条：实时显示处理进度
