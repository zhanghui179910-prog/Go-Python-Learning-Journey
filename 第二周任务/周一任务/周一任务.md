💻 周一：Goroutine 与 sync.WaitGroup 基础实战
题目：并发文件读取性能测试

目标：掌握 Goroutine 的基本用法，理解并发执行的优势，学会使用 sync.WaitGroup 协调多个 Goroutine 的同步执行。

## 需求说明

Go 语言的并发模型是其核心优势之一。今天我们将学习 Go 并发编程的基础：**Goroutine（协程）** 和 **sync.WaitGroup（等待组）**。

请实现一个**并发文件读取性能测试程序**，该程序能够：

### 1. 准备工作
- 创建 100 个测试文件，文件名格式为 `file_0.txt` 到 `file_99.txt`
- 每个文件包含简单的文本内容："这是第 X 个文件的内容"

### 2. 并发读取设计
- 使用 `sync.WaitGroup` 协调 100 个 Goroutine 的并发执行
- 每个 Goroutine 负责读取一个文件
- 主程序等待所有 Goroutine 完成工作

### 3. 性能对比
- 记录并发读取的总耗时
- （可选）与串行读取方式对比性能差异

### 4. 清理工作
- 处理完成后删除所有测试文件
- 使用 `defer` 确保异常情况下也能清理资源

## 技术要点

### Goroutine 启动
```go
// 启动一个 Goroutine
go func() {
    // 并发执行的代码
}()

// 带参数启动 Goroutine（避免闭包陷阱）
go func(id int) {
    // 使用参数 id 而不是循环变量 i
}(i)
```

### sync.WaitGroup 使用
```go
var wg sync.WaitGroup

// 正确顺序：先 Add，后启动 Goroutine
wg.Add(1)  // 增加等待计数
go func() {
    defer wg.Done()  // 确保 Done 被调用
    // 工作逻辑
}()

// 等待所有 Goroutine 完成
wg.Wait()
```

### 文件操作
```go
// 创建目录
os.MkdirAll("./test_files", 0755)

// 写入文件
os.WriteFile(fileName, []byte(content), 0644)

// 读取文件
data, err := os.ReadFile(fileName)
```

### 路径处理（跨平台）
```go
// 使用 filepath.Join 而不是字符串拼接
fileName := filepath.Join(tempDir, fmt.Sprintf("file_%d.txt", id))
```

## 关键难点与解决方案

### 1. WaitGroup 使用顺序
```go
// ❌ 错误：Add 在 Goroutine 内部
go func() {
    wg.Add(1)  // 可能导致主程序提前 Wait()
    defer wg.Done()
}()

// ✅ 正确：Add 在 Goroutine 外部
wg.Add(1)
go func() {
    defer wg.Done()
}()
```

### 2. 闭包变量捕获问题
```go
// ❌ 错误：所有 Goroutine 可能读取到相同的 i 值
for i := 0; i < 100; i++ {
    go func() {
        fmt.Println(i)  // 可能全部打印 100
    }()
}

// ✅ 正确：传递参数副本
for i := 0; i < 100; i++ {
    go func(id int) {
        fmt.Println(id)  // 正确打印 0-99
    }(i)
}
```

### 3. 资源清理保障
```go
// 使用 defer 确保清理执行
defer os.RemoveAll(tempDir)
defer wg.Done()  // 确保计数器减一
```
