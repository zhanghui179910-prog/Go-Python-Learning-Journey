题目：生产者-消费者模型实战

目标：掌握有缓冲通道的基本用法，理解 Goroutine 间的通信机制，实现简单的生产者-消费者模式。

## 需求说明

基于周一学习的 Goroutine 和 WaitGroup 基础，今天将学习 Go 并发编程的核心组件：**通道（Channel）**。通道是 Goroutine 之间进行通信和数据交换的安全管道。

请实现一个**简单的生产者-消费者模型**，该程序能够：

### 1. 生产者协程
- 生成 1-15 的数字序列
- 将每个数字发送到有缓冲通道（容量为5）
- 发送完成后关闭通道

### 2. 消费者协程
- 从通道接收数字
- 计算每个数字的平方
- 模拟处理耗时（每秒处理一个数字）

### 3. 程序控制
- 使用 `sync.WaitGroup` 等待两个协程完成
- 主程序在协程完成后安全退出

## 技术要点

### 通道（Channel）操作
```go
// 创建有缓冲通道，容量为5
numbers := make(chan int, 5)

// 发送数据到通道
numbers <- i

// 从通道接收数据
num := <-numbers

// 关闭通道（由发送方关闭）
close(numbers)

// 使用 range 遍历通道（自动检测关闭）
for num := range numbers {
    // 处理数据
}
```

### Goroutine 与 WaitGroup 配合
```go
var wg sync.WaitGroup

// 生产者协程
wg.Add(1)
go func() {
    defer wg.Done()
    // 生产逻辑
}()

// 消费者协程
wg.Add(1)
go func() {
    defer wg.Done()
    // 消费逻辑
}()

// 等待所有协程完成
wg.Wait()
```

## 预期学习成果

1. **理解通道的本质**：通道是类型安全的 FIFO 队列，用于 Goroutine 间通信
2. **掌握缓冲通道**：缓冲通道允许异步发送，当缓冲区满时发送方会阻塞
3. **学会生产者-消费者模式**：基本的并发协作模式
4. **实践通道关闭机制**：正确的通道关闭时机和方式
5. **理解通道遍历**：使用 `range` 自动接收直到通道关闭
