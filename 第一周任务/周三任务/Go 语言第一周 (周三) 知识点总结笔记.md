1. 抛弃 Class，拥抱 Struct（结构体）
核心概念：Go 语言中没有 class 和继承的概念。一切面向对象的实体（如用户、题库、电子元器件），都通过定义 struct（结构体）来实现。

本质：Struct 就像是一张“图纸”，里面把相关联的数据（属性）捆绑在一起。

实例化：用图纸造出实体的过程。例如 r := Resistor{Name: "R1", Value: 220.0}。



2. 方法 (Method)：被“圈养”的函数
语法结构：func (接收者变量名 接收者类型) 方法名(参数) 返回值 { ... }

接收者 (Receiver)：绑定在函数名前面的“紧箍咒”，相当于 Python 里的 self。它明确规定了这个函数只能被特定的结构体调用。

调用方式：通过点操作符调用实体里的数据或方法，例如 r.Value（获取属性）或 r.GetPower(2.0)（调用方法）。



3. 终极天堑：值接收者 vs 指针接收者 (*)
值接收者 (u User)（拿复印件）：

Go 语言会把结构体完完整整复制一份交给方法。

特点：在方法内做的任何修改，改的都是“复印件”，原件毫发无损。适用于只读操作或纯计算（如算功耗）。

指针接收者 (u *User)（拿门牌号/原件）：

加上 * 后，传给方法的是该实体在内存中的“真实地址”。

特点：方法可以顺着地址找到原件，直接修改底层真实数据。只要涉及状态更新（如增加打卡天数），必须使用指针！
