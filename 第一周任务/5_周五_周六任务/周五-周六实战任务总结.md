1. 项目目标：构建一个具备持久化能力的 CLI 任务管理工具 。

2. 核心逻辑链条：


启动阶段：调用 loadTasks()，尝试从 tasks.json 读取历史数据并填充到 memoryTasks 切片 。


交互阶段：通过 switch 匹配 os.Args[1]，分发出 add、list、done、delete、add-sub 等动作 。

内存操作：


增/改：直接对 memoryTasks 或其索引元素进行操作 。


删：利用 append(tasks[:i], tasks[i+1:]...) 魔法公式实现元素剔除 。


终结阶段：每次修改数据后，必须调用 saveTasks() 将最新的内存快照写回硬盘 。

3. 避坑指南（复盘博客核心素材）：


索引 0 陷阱：os.Args[0] 是程序路径，真正的参数从 1 开始 。


修改失效：在 for range 循环中，必须通过 tasks[i].Status 修改，直接改临时变量 task.Status 无效 。


程序失忆：如果忘了在方法末尾调用 saveTasks，程序一关，刚才的操作全部白费 。
