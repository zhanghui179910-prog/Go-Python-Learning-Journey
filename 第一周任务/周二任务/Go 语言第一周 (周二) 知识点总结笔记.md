一、 映射 (Map) 的“生存法则”
Map 是 Go 里的“字典”，它的设计哲学是 “严进宽出”：

严进（初始化）：永远不要忘记 make(map[keyType]valueType)。没有 make 的 Map 是 nil，强行写入会导致程序崩溃。

宽出（零值特性）：这是 Go 最温柔的地方。访问不存在的键不会报错（不像 Python 会炸），而是返回该类型的零值（int 返回 0，string 返回 ""）。

计数魔法：m[key]++ 这一行代码包含了“查找 -> 赋初值0 -> 加1 -> 存回”的全过程。



二、 切片 (Slice) 的“扩容奥秘”
切片不是数组，它是数组的窗口：

切片 vs 数组：[4]int 是死板的水泥盒子（数组）；[]int 是会伸缩的皮筋（切片）。

扩容 (Cap)：当 append 发现底层数组装不下时，Go 会申请一个更大的新数组（通常是原来的 2倍），并将老数据搬过去。

观察结论：cap 的跳跃式增长（1→2→4→8）是为了减少内存申请次数，提高性能。



三、 数据形态转换流水线
为什么一个简单的统计要写几十行？因为数据在改变形状：

Split 阶段：string ➔ []string (把杂乱的日志切开)。

Map 阶段：[]string ➔ map[string]int (为了利用哈希表飞速数数)。

Struct 阶段：map ➔ []struct (因为 Map 无序，必须搬家到切片队列里才能排序)。

Sort 阶段：使用 sort.Slice 配合“临时裁判”规则进行排队。



四、 避坑小贴士 (Today's Lessons)

下划线 _：它是 Go 的“垃圾桶”，专门回收你定义了却不想用的变量（比如 for range 里的索引）。

Printf 里的 f：fmt.Printf 才有格式化功能，Print 只能印原样。

排序裁判：func(i, j int) bool 只是在回答 Go 的一个问题：“第 i 个元素应该排在第 j 个前面吗？”
